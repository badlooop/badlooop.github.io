Run ROS on Docker

發表於 2015-11-01

前言




因为最近涉及到大量的嵌入式开发以及ROS机器人平台的开发，开始尝试将最近火热的Docker技术引入嵌入式和机器人开发中来。Docker容器具有良好的可迁移性及极高的性能，又能在现有的系统上起到一定的隔离作用。

嵌入式开发：

  • 工具链配好，commit一次便可一劳永逸，换电脑亦可继续使用
  • 良好地保护了宿主系统，完全防止开发人员忘记修改PREFIX环境变量，运行make install之后导致的系统崩溃
  • 可以利用Docker Hub进行工具链共享

ROS：

  • 良好的迁移性让开发者可以在不同机器人所搭载的PC上共享同样的ROS环境（当然，宿主系统得相同）
  • 多个版本的ROS可以各在一个容器中，互不干涉
  • IO性能远高于虚拟机，特别是对于RGBD摄像头等大数据量的硬件设备来说，Docker能提供和原生系统一致的IO性能
  • （不确定）能支持GPU，对于CUDA用户和深度学习玩家来说将会非常有用 （StackoverFlow说支持，暂时未能测试）

安装Docker

由于众所周知的原因，Docker官方所在的Amazon S3服务器不容易访问。我们可以使用国内一个替代的源，由DaoCloud.io所提供

curl -sSL https://get.daocloud.io/docker | sh

Docker的简单入门教程在 [http://docker.widuu.com]

Compiling the RT3070 driver for Hi3516D

發表於 2015-10-30

It is a little bit tricky to cross-compile Hi3516d driver for Hi3516d. The following are some key modifications.

PREREQUISITE:

  • Hi3516D SDK
  • RT3070 Linux Driver (in source code)

Key steps:

Modify kernel configuration file:

  • Setup cross-compiling environment, including PATH
  • Compiling osdrv folder under

  • Modify the Makefile in osdrv folder

    Delete these two lines:

    rm $(OSDRV_DIR)/opensource/kernel/linux-3.4.y -rf
tar xzf $(OSDRV_DIR)/opensource/kernel/linux-3.4.y.tgz -C $(OSDRV_DIR)/opensource/kernel/

    Because we need to modify the kernel compiling configuration file to support USB wlan driver, we have to toggle off these two lines.
    Modify the KERNEL_CFG to hi3516a_full_defconfig
    export KERNEL_CFG:=hi3516a_full_defconfig

  • Change the kernal compiling configuration

    cd #rm $(OSDRV_DIR)/opensource/kernel/linux-3.4.y
    make menuconfig
    load the configuration file from /arch/arm/configs/hi3516a_full_defconfig
    go to Device Driver -> Network device support -> Wireless LAN, and enable the USB ZD1201 based Wireless device support
    save the configuration file

  • re-compile the osdrv

    cd $(OSDRV_DIR)
    make OSDRV_CROSS=arm-hisiv300-linux all

Compile RT3070 driver

  • Decompress tar.gz file
  • Change the CHIPSET to 3070

    CHIPSET = 3070

  • Create a new platform: Hi3516d

    ifeq ($(PLATFORM),Hi3516d)
    LINUX_SRC = ~/Hi3516A_SDK_040/osdrv/opensource/kernel/linux-3.4.y
    CROSS_COMPILE = arm-hisiv300-linux-

  • Modify the $(RT28xx_DIR)/os/linux/config.mk
    Here is the most tricky part. You will have to use -D__LINUX_ARM_ARCH__=7 -march=armv7-a to get right binary format, and use -mabi=aapcs-linux -mfloat-abi=soft to use EABI instead of OABI. You can find a good reference [here][0].
    Otherwise you will get:

    Unable to handle kernel NULL pointer dereference at virtual address 00000028
    config.mk:
    Enable the WPA supplicant:
    HAS_WPA_SUPPLICANT=y
    HAS_NATIVE_WPA_SUPPLICANT_SUPPORT=y
    and add these lines:
    ifeq ($(PLATFORM),Hi3516d)
    #WFLAGS += -DRT_BIG_ENDIAN
    endif

    ifeq ($(PLATFORM),Hi3516d)
    EXTRA_CFLAGS := -v $(WFLAGS) -I$(RT28xx_DIR)/include -DLINUX_ARM_ARCH=7 -march=armv7-a -mabi=aapcs-linux -mfloat-abi=soft
    export EXTRA_CFLAGS
    endif

  • Make it!

    make ARCH=arm

  • Copy the rt3070sta.ko file to your board.

  • Copy the RT2870STA.dat to /etc/Wireless/RT2870STA/

  • Insert the kernal module

    insmod rt3070sta.ko

  • Done!

Ref

[0]: http://blog.chinaunix.net/uid-9672747-id-4779324.html ‘rt3070在dm365上的移植’
